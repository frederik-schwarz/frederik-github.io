Rubber duck method

The rubber duck method is the process of explaining your problem to a rubber duck. This works because it allows us to take a step back and look at the bigger picture. When we explain a problem we need to give context and often while we are giving this context the solution will come to us.

Pseudocode:

Pseudocode is an extremely helpful planner tool to help you avoid errors and problems before they arise. The keys to pseudocode are single-line comments, descriptive comments with keywords such as IF and Loop. Pseudocode is something that I think can help me a lot and I should use.

The simple problem I got stuck on:

While working through replit I got stuck on the built-in methods section. Built-in methods are pre-written pieces of code that you can access and use through a keyword. These built-in methods come with a general structure. The structure includes a function parameter and a follow-on event. It took me a long time to figure out this structure and understand how methods work. lso struggled with accessing these methods once I used them. How did I call them into action? The key to this process is naming the var that you have linked to it and follow this with the parameters in brackets.
